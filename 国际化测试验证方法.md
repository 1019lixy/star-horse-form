# 国际化测试验证方法

## 目的
本文档旨在提供一套完整的测试和验证方法，确保组件在发布后国际化内容能正常显示。

## 测试验证场景

### 1. 初始加载测试

**测试要点：**
- 验证组件在初始加载时，国际化内容是否正确显示
- 验证在不同语言环境下，组件的默认显示是否符合预期

**测试步骤：**
```bash
# 构建项目
npm run build

# 启动服务
npm run dev

# 手动测试不同语言环境下的初始显示
```

**自动化测试示例：**
```javascript
// 组件初始加载测试
import { mount } from '@vue/test-utils';
import { loadLanguage } from '@/lang';
import MyComponent from './MyComponent.vue';

describe('组件初始加载国际化测试', () => {
  it('中文环境下应显示正确的中文文本', async () => {
    await loadLanguage('zh-CN');
    const wrapper = mount(MyComponent);
    
    // 验证特定的中文文本是否存在
    expect(wrapper.text()).toContain('正确的中文文本');
  });
  
  it('英文环境下应显示正确的英文文本', async () => {
    await loadLanguage('en-US');
    const wrapper = mount(MyComponent);
    
    // 验证特定的英文文本是否存在
    expect(wrapper.text()).toContain('Correct English Text');
  });
});
```

### 2. 语言切换测试

**测试要点：**
- 验证组件在语言切换时，国际化内容是否能实时更新
- 验证配置中的国际化内容是否能响应语言变化

**测试步骤：**
1. 打开应用，选择一种语言
2. 验证组件显示正确的内容
3. 切换到另一种语言
4. 验证组件内容是否正确更新

**自动化测试示例：**
```javascript
// 语言切换测试
import { mount } from '@vue/test-utils';
import { loadLanguage, changeLanguage } from '@/lang';
import MyComponent from './MyComponent.vue';

describe('组件语言切换测试', () => {
  let wrapper;
  
  beforeEach(async () => {
    await loadLanguage('zh-CN');
    wrapper = mount(MyComponent);
  });
  
  it('从中文切换到英文时应更新文本', async () => {
    // 初始状态验证
    expect(wrapper.text()).toContain('中文文本');
    
    // 切换语言
    await changeLanguage('en-US');
    
    // 验证切换后文本
    expect(wrapper.text()).toContain('English Text');
    expect(wrapper.text()).not.toContain('中文文本');
  });
});
```

### 3. 延迟加载测试

**测试要点：**
- 验证在国际化内容延迟加载的情况下，组件是否能正确显示
- 验证配置函数是否能在语言包加载完成后正确执行

**测试步骤：**
1. 模拟缓慢的语言包加载
2. 验证组件是否能正确等待并显示内容

**自动化测试示例：**
```javascript
// 延迟加载测试
import { mount } from '@vue/test-utils';
import { vi, Mock } from 'vitest';
import { loadLanguage } from '@/lang';
import MyComponent from './MyComponent.vue';

// 模拟延迟加载
vi.mock('@/lang', () => ({
  loadLanguage: vi.fn().mockImplementation(() => new Promise(resolve => {
    setTimeout(resolve, 100); // 模拟100ms延迟
  })),
  changeLanguage: vi.fn(),
  i18n: vi.fn(key => key) // 简单返回键名用于测试
}));

describe('组件延迟加载测试', () => {
  it('应等待语言包加载完成后显示内容', async () => {
    const wrapper = mount(MyComponent);
    
    // 初始状态可能为空或占位符
    expect(wrapper.text()).not.toContain('最终显示的文本');
    
    // 等待语言包加载
    await new Promise(resolve => setTimeout(resolve, 150));
    
    // 验证加载后的内容
    expect(wrapper.text()).toContain('最终显示的文本');
  });
});
```

### 5. 性能测试

**测试要点：**
- 验证国际化内容处理对组件渲染性能的影响
- 确保响应式配置不会导致不必要的重新渲染

**测试步骤：**
```javascript
// 性能测试示例
import { mount } from '@vue/test-utils';
import { changeLanguage } from '@/lang';
import MyComponent from './MyComponent.vue';

describe('组件性能测试', () => {
  it('语言切换时不应该导致过多的重新渲染', () => {
    const renderCount = ref(0);
    const wrapper = mount(MyComponent, {
      global: {
        provide: {
          // 提供一个计数器来监控渲染次数
          renderTracker: {
            increment: () => renderCount.value++
          }
        }
      }
    });
    
    // 切换语言并验证渲染次数
    changeLanguage('en-US');
    expect(renderCount.value).toBeLessThan(3); // 应该在合理范围内
  });
});
```

## 常见问题排查

### 1. 初始加载时显示键名而非翻译文本

**可能原因：**
- 语言包加载顺序问题
- 配置在国际化初始化前执行

**解决方案：**
- 确保使用函数形式的配置
- 使用`ensureI18nReady`工具函数

### 2. 语言切换后内容不更新

**可能原因：**
- 没有监听语言变化事件
- 配置不是响应式的

**解决方案：**
- 使用`createReactiveI18nConfig`创建响应式配置
- 实现语言变化监听逻辑

### 3. 构建后显示异常

**可能原因：**
- 构建配置问题
- 动态导入的语言包未正确打包

**解决方案：**
- 检查构建配置中的动态导入设置
- 确保语言包文件被正确包含在构建产物中
